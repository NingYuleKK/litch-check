**最后更新时间:** 2026年2月23日

**文档作者:** Manus AI

## 1. 项目概览

- **项目名称:** Litch's Check
- **项目简介:** 一款个人目标反馈与习惯追踪工具，旨在通过游戏化的方式帮助用户保持动力，完成每日设定的目标。
- **核心理念:** 结合积极心理学与趣味性设计，将日常任务转化为富有成就感的体验。

## 3. 功能列表

### 3.1 每日打卡系统

- **五类固定任务:**
  - 包含"学英语"、"读书"、"AI/AIGC"、"锻炼"、"功课与正念"五个核心任务。
  - 每个任务都有专属的颜色和手绘风格图标，便于视觉区分。

- **每日打卡:**
  - 用户可以为每个完成的任务勾选"完成"。
  - 支持为每次打卡添加简短的文字记录或心得。

- **成就系统:**
  - **星星成就:** 当天完成 ≥3 个任务时，获得"猫竖大拇指"图标。
  - **大圆满成就:** 当天完成所有 5 个任务时，获得"猫抱莲花"图标。

- **司马黑（胖黑猫吉祥物）:**
  - 根据用户的打卡状态，吉祥物会切换四种不同的形态（Normal / Encourage / Perfect / Lazy）。
  - 带有贱萌风格的对话气泡，根据完成状态和连胜天数动态生成。

- **连胜天数追踪:**
  - 自动记录用户连续打卡的天数。
  - 在达到特定的里程碑（如 3 天、7 天、30 天）时，会发送祝贺消息。

### 3.2 数据可视化

- **月日历视图:** 在日历上清晰展示每日的成就状态（无标记、星星或莲花）。
- **每周完成率:** 以柱状图的形式展示最近一周内每日任务的完成比例。
- **月度统计:** 汇总展示当月的总打卡天数、各项成就获得次数等数据。

### 3.3 日记与回顾

- 提供一个独立的每日日记/Log 区域，供用户记录详细感想。
- 用户可以编辑和删除过去的打卡记录和日记。

### 3.4 周报结算系统 (v2.0 新增)

- **自动生成周报:**
  - 基于每周一到周日的打卡数据，自动计算并生成结算单。
  - 无需额外的数据库表，直接从 `checkins` 表聚合查询。

- **结算单内容:**
  - **星星天数:** 当天完成 ≥3 个任务的天数（满分 7）。
  - **大圆满天数:** 当天完成所有 5 个任务的天数（满分 7）。
  - **5 个分类各自的完成情况:** 每个分类本周完成次数 / 7 天。
  - **司马黑评语:** 根据本周表现动态生成的贱萌风格一句话评语。

- **猫评语规则:**

| 表现等级 | 条件 | 评语风格 |
| :--- | :--- | :--- |
| 全勤周 (perfect) | 7 天都 ≥3 | "这周你比我还勤快，不科学。" |
| 大圆满多 (lotusMany) | ≥3 天全 5 | "莲花都快被你摘完了，佛祖要找你谈话。" |
| 一般般 (decent) | 4-6 天 ≥3 | "还行吧，勉强配得上我的陪伴。" |
| 摸鱼周 (slacking) | 1-3 天 ≥3 | "这周你是不是把我忘了？" |
| 完全没打卡 (absent) | 0 天 ≥3 | "……你还活着吗？" |

- **周一弹窗提醒:**
  - 每周一打开网站时，自动弹窗提醒用户查看上周周报。
  - 使用 `localStorage` 记录已查看状态，避免重复弹窗。

- **历史周报列表:**
  - 底部导航栏新增"周报"入口，可随时查看历史周报。
  - 列表按时间倒序排列，显示每周的星星天数和大圆满天数。
  - 当前周（还没到周一）显示"待结算"状态。

- **导出图片:**
  - 使用 `modern-screenshot` 库（`domToBlob`）将结算卡片导出为 PNG 图片。
  - 2x 分辨率，保留孟菲斯风格装饰和猫图片。

## 4. 设计风格

- **孟菲斯设计 (Memphis Design):**
  - **主色调:** 采用柔和的桃色 (#FFB5A7) 作为背景主色调。
  - **装饰元素:** 页面中会随机浮动一些简单的几何图形（如圆形、三角形、波浪线），增加页面的趣味性和活力。
  - **周报卡片:** 采用渐变背景（#FFF5F0 → white → #F0E6FF），搭配孟菲斯风格的圆形、三角形、菱形装饰。

## 6. API 端点

### 打卡相关

| 端点 | 方法 | 说明 |
| :--- | :--- | :--- |
| `trpc.checkin.getByDate` | Query | 获取指定日期的打卡数据 |
| `trpc.checkin.toggle` | Mutation | 切换任务完成状态 |
| `trpc.checkin.updateNote` | Mutation | 更新任务备注 |
| `trpc.checkin.deleteNote` | Mutation | 删除任务备注 |
| `trpc.checkin.monthlySummary` | Query | 获取月度汇总 |
| `trpc.checkin.streak` | Query | 获取连胜天数 |

### 周报相关 (v2.0 新增)

| 端点 | 方法 | 说明 |
| :--- | :--- | :--- |
| `trpc.weeklyReview.get` | Query | 获取指定周的结算数据（输入: weekStart "YYYY-MM-DD"） |
| `trpc.weeklyReview.list` | Query | 获取所有历史周报列表 |

### 日记相关

| 端点 | 方法 | 说明 |
| :--- | :--- | :--- |
| `trpc.dailyLog.get` | Query | 获取指定日期的日记 |
| `trpc.dailyLog.save` | Mutation | 保存/更新日记 |

## 7. 数据库表

### checkins 表

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | INT (PK) | 自增主键 |
| dateStr | VARCHAR(10) | 日期 "YYYY-MM-DD" |
| taskId | INT | 任务 ID (1-5) |
| completed | BOOLEAN | 是否完成 |
| note | TEXT | 备注（可选） |
| createdAt | TIMESTAMP | 创建时间 |
| updatedAt | TIMESTAMP | 更新时间 |

### daily_logs 表

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | INT (PK) | 自增主键 |
| dateStr | VARCHAR(10) | 日期 "YYYY-MM-DD" |
| content | TEXT | 日记内容 |
| createdAt | TIMESTAMP | 创建时间 |
| updatedAt | TIMESTAMP | 更新时间 |

> **注意:** 周报数据不需要额外的数据库表，直接从 `checkins` 表按日期范围聚合查询。

## 9. 后续维护建议

- **数据备份:** 建议定期对数据库进行备份，以防数据丢失。
- **扩展新任务:** 未来如果需要增加新的任务类型，需要在 `shared/tasks.ts` 中添加新的任务定义，准备并上传新的手绘图标到 S3 CDN，并更新相关的阈值常量。
- **周报增强:** 可以考虑添加月报/季报功能，或者在周报中加入趋势对比（与上周相比）。
- **社交分享:** 导出的周报图片可以进一步集成微信/微博等社交平台的分享功能。
- **用户反馈:** 建议增加一个简单的用户反馈渠道，以便收集用户意见，持续改进产品。
